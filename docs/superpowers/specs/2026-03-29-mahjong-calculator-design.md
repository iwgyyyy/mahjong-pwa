# 立直麻将番符计算器 — 设计文档

## 概述

一个 PWA 番符计算器工具，用于线下立直麻将对局中快速计算手牌的役种、番数、符数和点数。图形化选牌输入，支持移动端和 PC 端。

## 需求总结

| 项目 | 决定 |
|------|------|
| 手牌输入 | 图形化选牌（SVG 牌面） |
| 副露输入 | 分区输入，先选类型（吃/碰/明杠/暗杠）再选牌 |
| 和了牌 | 单独输入框，额外选一张 |
| 场况条件 | 同页面，开关面板 |
| 结果展示 | 役种列表 + 番符 + 符数详细计算过程 + 点数 |
| 多种解释 | 自动选点数最高的解释 |
| UI 风格 | 简洁工具风，shadcn/ui |
| 牌面显示 | SVG 素材（FluffyStuff/riichi-mahjong-tiles） |
| 响应式 | 移动端 + PC 端 |

## 页面结构

单页长滚动布局，从上到下依次为：

1. **场况条件栏** — 场风、自风、本场数、特殊条件开关
2. **手牌区（门前）** — 已选手牌，点击可移除
3. **副露区** — 已添加的副露组合 + "添加副露"按钮
4. **和了牌区** — 单张牌 + 自摸/荣和切换
5. **选牌键盘** — 万/筒/索/字四行 SVG 牌面
6. **计算结果区** — 役种列表、番数、符数详细过程、最终点数

### 响应式策略

| | 移动端（< 768px） | PC 端（>= 768px） |
|---|---|---|
| 布局 | 单列，全部纵向堆叠 | 双列：左侧手牌/副露/和了牌，右侧条件+结果 |
| 选牌键盘 | 固定在屏幕底部（类似手机键盘） | 内联显示在手牌区下方 |
| 牌面大小 | 较大触摸目标（约 40×56px） | 标准大小（约 36×48px） |

## 手牌输入与副露交互

### 手牌区（门前）

- 显示为一排 SVG 牌面，最多 13 张（无副露时），随副露递减
- 点击已选牌面可移除
- 空位用虚线占位符提示还能选几张
- 张数自动计算：13 - (副露组数 × 3)，杠为 13 - (副露组数 × 3) - 额外杠张数

### 副露区交互流程

1. 点击"+ 添加副露"按钮
2. 弹出类型选择：吃 / 碰 / 明杠 / 暗杠（shadcn Popover 或 DropdownMenu）
3. 选择类型后，副露区进入"活跃状态"，从键盘选牌：
   - **吃**：选 3 张不同的牌（系统校验是否为顺子）
   - **碰**：选 1 张牌，自动生成 3 张相同
   - **明杠**：选 1 张牌，自动生成 4 张相同
   - **暗杠**：选 1 张牌，自动生成 4 张相同（显示背面朝上样式以区分）
4. 每组副露显示为带边框的 card，右上角有删除按钮

### 选牌键盘

- 四行：万子(1-9m)、筒子(1-9p)、索子(1-9s)、字牌(东南西北白发中)
- 每张牌显示剩余可选数量（总 4 张减去已选张数），为 0 时灰显不可点
- 点击即添加到当前活跃区域（手牌区/副露区/和了牌区）

### 焦点机制

用户点击手牌区/副露区/和了牌区时，该区域高亮为"活跃状态"，此时从键盘选牌会添加到对应区域。

## 场况条件

### 基本场况

- **场风**：东 / 南（Toggle Group，二选一）
- **自风**：东 / 南 / 西 / 北（Toggle Group，四选一）；选"东"时自动标记为庄家（亲），其余为子家
- **本场数**：数字输入（0, 1, 2, 3...），影响场供计算（每本场 300 点）

### 特殊条件开关（Toggle 按钮，可多选）

- 立直 / 双立直（互斥）
- 一发（依赖立直开启）
- 门前清自摸（自动判定：门前 + 自摸时自动勾选，不可手动）
- 岭上开花
- 抢杠
- 海底摸月 / 河底捞鱼（互斥，且与自摸/荣和联动）
- 天和 / 地和（极少用，折叠在"更多"里）

### 条件联动逻辑

- 选"自摸"→ 海底摸月可选，河底捞鱼不可选
- 选"荣和"→ 河底捞鱼可选，海底摸月不可选
- 有副露 → 立直/一发/双立直不可选
- 立直关闭 → 一发自动关闭

### 和了牌区

- 高亮的单牌槽位，点击后激活选牌键盘
- 旁边是自摸/荣和的 Toggle 切换（shadcn Toggle Group）

## 计算引擎

### 触发机制

- 手牌 + 和了牌输入完成后自动计算（无需点击按钮）
- 输入不完整时，结果区显示提示（如"请选择和了牌"）

### 计算流程

1. **面子拆解** — 枚举所有合法的手牌拆解方式（顺子/刻子/雀头），包括七对子、国士无双等特殊牌型
2. **役种判定** — 对每种拆解，检测所有成立的役（含场况条件）
3. **符数计算** — 对每种拆解计算符数：
   - 底符 20（七对子固定 25 符）
   - 门前荣和 +10
   - 自摸 +2（平和自摸例外为 0）
   - 雀头：役牌雀头 +2
   - 面子：明顺 0 / 暗顺 0 / 明刻 2-4 / 暗刻 4-8 / 明杠 8-16 / 暗杠 16-32（根据中张/幺九区分）
   - 待ち形：两面 0 / 嵌张 +2 / 边张 +2 / 双碰 0 / 单骑 +2
   - 切上：30 符以上十位进位
4. **选最优解** — 所有拆解中取点数最高的结果
5. **点数查表** — 番数 × 符数 → 基本点，再根据亲/子、自摸/荣和计算实际点数

### 结果展示（三层）

```
┌─────────────────────────────────┐
│  满贯  7700 点（子家荣和）        │  ← 第一层：总结（始终显示）
├─────────────────────────────────┤
│  役种：                          │  ← 第二层：役种明细（默认展开）
│    立直     1番                  │
│    平和     1番                  │
│    断幺九   1番                  │
│  ──────────────────             │
│  合计：3番 30符                  │
├─────────────────────────────────┤
│  符数计算：                      │  ← 第三层：符数详情（默认折叠）
│    底符           20             │
│    门前荣和       +10            │
│    雀头(无役牌)    +0            │

```

## 技术架构

### 技术栈

- Next.js 16 + React 19 + TypeScript
- Tailwind CSS 4 + shadcn/ui
- 纯前端计算，无后端

### 路由结构

- `/` — 首页（导航入口）
- `/calculator` — 番符计算器页面

### 组件拆分

```
app/calculator/page.tsx          — 页面入口，组合各区域
components/calculator/
  ├── GameConditions.tsx          — 场况条件栏（场风、自风、本场数、特殊条件）
  ├── HandTiles.tsx               — 手牌区（门前）
  ├── MeldSection.tsx             — 副露区（含添加副露交互）
  ├── WinningTile.tsx             — 和了牌 + 自摸/荣和
  ├── TileKeyboard.tsx            — 选牌键盘
  ├── ResultPanel.tsx             — 计算结果展示
  └── TileImage.tsx               — 单张牌的 SVG 渲染组件
lib/mahjong/
  ├── types.ts                    — 类型定义（牌、面子、役种等）
  ├── parser.ts                   — 手牌拆解（面子枚举）
  ├── yaku.ts                     — 役种判定
  ├── fu.ts                       — 符数计算
  ├── score.ts                    — 点数查表
  └── calculator.ts               — 计算入口（组合以上模块）
```

### 状态管理

- 用 React `useState` / `useReducer` 在 `page.tsx` 层级管理
- 状态结构：当前活跃区域、手牌数组、副露数组、和了牌、场况条件
- 不需要全局状态库

### 数据持久化

- `localStorage` 保存当前输入状态，防止误刷新丢失
- "清空"按钮一键重置

### SVG 素材

- FluffyStuff/riichi-mahjong-tiles 下载到 `public/tiles/`
- `TileImage` 组件根据牌 ID 加载对应 SVG
